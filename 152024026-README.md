## Deskripsi Sistem
Sistem ini mensimulasikan deteksi pelanggaran batas kecepatan pada jalan raya (Speed Camera System). Untuk mengoptimalkan performa dan menghindari *delay* pemrosesan data visual, sistem ini menerapkan **Komputasi Paralel dengan metode Forking/Multiprocessing**.

Sistem dibagi menjadi 2 proses utama yang berjalan secara independen secara bersamaan:
1. **Proses Kamera (Producer):** Berfungsi menangkap atau mensimulasikan input data kendaraan yang melintas secara kontinu.
2. **Proses Deteksi Pelanggaran (Consumer):** Menerima data dari proses kamera secara real-time melalui IPC (Inter-Process Communication) Queue dan menyaring kendaraan yang melebihi batas kecepatan 80 km/jam untuk ditilang.

## Konsep Komputasi Paralel yang Diterapkan
- **Forking/Multiprocessing:** Memisahkan beban kerja I/O bound (kamera) dan CPU bound (analisis data kecepatan) ke Core prosesor yang berbeda.
- **Inter-Process Communication (IPC):** Menggunakan `multiprocessing.Queue` yang aman (*thread/process-safe*) untuk mengirim data dari Proses 1 ke Proses 2.
